## 1. Release Overview

This audit covers the period **since the v1.1 audit closed (Apr 14)** through the `fe/v1.2.0` tag (Jun 5). It is deliberately scoped to *new* work — the bulk of the commits reachable from `fe/v1.2.0` predate Apr 15 and were already accounted for in the v1.1 audit (the `fe/v1.1.0` tag was cut Mar 27, but FE development continued through mid-April under that audit window).

**v1.2 is a consolidation and release-automation cycle, not a feature cycle.** After the intense v1.1 push (134 issues, 354 story points in 28 days), v1.2 is light and focused: operationalize iOS delivery, add product telemetry, swap the SAST tool, fix a Live Activity reliability bug, and keep dependencies current.

### Highlights

- **Automated TestFlight pipeline** — Fastlane (`match` + `pilot`) on GitHub Actions; first proven end-to-end run **2026-06-05** (build `1.2.0 (21)`). Closes the v1.1 audit's #9 action item. See [ADR-041](../adr/041-automated-testflight-fastlane.md).
- **Product telemetry** — Firebase Analytics added (`app_open` + custom `timer_session_completed`). See [ADR-040](../adr/040-firebase-analytics-telemetry.md).
- **SAST migration** — CodeQL/GHAS replaced by Semgrep free tier (~$360/yr saved, ~30s vs 3–5 min CI). See [ADR-039](../adr/039-semgrep-over-codeql-for-sast.md).
- **Reliability fix** — Live Activity freeze on phase advance + `syncFromServer` cycle reset (MER-290).
- **Dependency upkeep** — multiple Dependabot bumps (`live_activities`, `sign_in_with_apple`, `package_info_plus`, `patrol`, `connectivity_plus`, …).
- **Release artifact:** `fe/v1.2.0`. No new backend release.

## 9. Executive Summary & Recommendations

### What Went Well

- **iOS delivery is now automated** — the single biggest operational gap from v1.1 is closed.
- **Product telemetry exists** — the app can finally answer "do users complete sessions."
- **Cheaper, faster security scanning** — Semgrep replaced CodeQL with no measured coverage loss for this codebase.
- **Dependencies kept current** — steady Dependabot hygiene.
- **Healthy backlog emerged** — the board now carries planned-but-not-started work instead of being drained to zero.

### What Needs Attention

- **FE tests are still non-blocking** (`continue-on-error: true`) — and 4 FE CI failures occurred this window. Highest-priority fix.
- **Six v1.1 tech-debt items carried over** untouched — notably the scheduler distributed lock (#6) and the `FocusSession.status` enum (#4).
- **No post-deploy verification** — still no smoke test (open since v1.0).
- **CI success dipped below v1.1's 100%** — partly masked by the non-blocking FE test step.

### Top Action Items for v1.3

| # | Action | Impact | Carry-over? |
| -- | -- | -- | -- |
| 1 | Remove `continue-on-error` from FE test step + fix the underlying failures | FE test failures currently invisible | v1.1 #1 |
| 2 | Add distributed lock to `SliceEndWarningScheduler` | Duplicate notifications on multi-instance | v1.1 #6 |
| 3 | Add post-deploy smoke test | Zero deployment verification | v1.0 #8 / v1.1 #2 |
| 4 | Refactor `FocusSession.status` to enum | Removes scattered string literals | v1.1 #4 |
| 5 | Fix `handleDataIntegrityViolation` differentiation | Wrong error codes for non-session conflicts | v1.1 #5 |
| 6 | Remove dead `SliceDateNotTodayException` | Hygiene | v1.1 #7 |
| 7 | Automate Android/Play delivery | iOS automated, Android still manual | New (ADR-041 follow-up) |
| 8 | Build the telemetry dashboard / define the events you actually need | Telemetry plumbed but under-used | New |

### Overall Project Health: STRONG (steady-state)

v1.2 is a healthy maintenance release: it closed the headline operational gap (iOS automation), added a missing capability (telemetry), and trimmed cost (Semgrep). The flip side is that the tech-debt backlog did not move and CI discipline slipped slightly. Neither is structural — but #1 (non-blocking FE tests) should not survive another release.
